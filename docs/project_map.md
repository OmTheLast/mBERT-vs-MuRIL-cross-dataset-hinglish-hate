# Project Map

Date: 2026-07-01

This file is the clean project map for GitHub and future paper work. It separates code, data, results, paper files, and research notes.

## Repository Purpose

This repository studies cross-dataset robustness of mBERT and MuRIL for Hinglish and Hindi-English code-mixed harmful speech detection.

The project started as a direct mBERT-vs-MuRIL comparison. It now asks a stronger question: how does the winner change across dataset situation, label policy, platform, script mix, and train/test domain?

## Source Of Truth Files

Use these files first when writing or defending the paper:

- `docs/dataset_registry.md`: dataset identity, label mapping, citation status, caveats.
- `docs/dataset_taxonomy.md`: how datasets differ by label, platform, script, and target.
- `docs/data_analysis_report.md`: dataset integrity and descriptive statistics.
- `docs/result_analysis_report.md`: primary model results and interpretation.
- `docs/error_analysis_report.md`: quantitative error analysis.
- `docs/manual_error_analysis_report.md`: first-pass qualitative error categories.
- `docs/project_defense_notes.md`: plain-language explanation for viva/project defense.
- `docs/research_rigor_roadmap.md`: what remains before final paper strength.

## Commit And Push Policy

From 2026-07-01 onward, significant project changes should be committed and pushed to GitHub in sequence.

Commit after:

- adding or converting a dataset;
- running a new experiment family;
- adding a new result table or figure;
- updating the paper draft materially;
- adding research-rigor or defense documentation;
- fixing reproducibility scripts.

Do not commit:

- raw datasets;
- processed datasets containing third-party text;
- model checkpoint weights;
- virtual environments;
- run caches;
- large prediction dumps.

Commit messages should describe the research step, not just the file edit.

Good examples:

- `Add research rigor roadmap and reproducible paper draft`
- `Run mixed-dataset transformer experiments`
- `Add multi-seed matched evaluation summary`
- `Document manual error examples for paper`
